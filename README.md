## 문제해결 방법과 순서

1. 팀원들 각자 코드 수정하고 commit 후 push하는 과정
2. 전원 수정 및 커밋, 푸쉬 완료
3. main 브랜치와 dev/a 브랜치 1차 병합 및 충돌 해결 
   - main 브랜치에서 dev/a 브랜치 병합을 시도했으나, 동일한 대상 파일`calc/calc.cpp`수정으로 인한 병합 충돌`Merge conflict`이 발생
   - 충돌 섹션을 수동으로 분석하여 유효 코드를 식별하고, sub.cpp 및 dive.cpp 내부 연산 로직 오류를 정상화한 후 충돌 해결 커밋을 완료
4. dev/b브랜치 병합 및 3-Way Merge 수행
5. dev/c 브랜치 최종 통합 및 2차 충돌 해결코드 수정 후 2번째 충돌 해결
6. 2번째 충돌 해결 후 git log
7. 마지막으로 브렌치 main에서 코드 실행시 발생하는 문제 확인 후 코드 수정 후 최종 커밋 및 푸쉬
8. 최종 실행 결과
